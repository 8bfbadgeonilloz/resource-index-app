# resource-index-app

这是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态 HTML 资源的索引与分发容器。

---

## 项目简介

`resource-index-app` 提供了一种轻量的方式，将多个独立的 HTML 页面集中存放，并通过 GitHub Pages 或其他静态托管服务直接访问。  
每个页面均为自包含的 HTML 文件，不依赖外部动态服务或后端逻辑。

---

## 目录结构

```
resource-index-app/
├── index.html          # 主入口页面（目录索引）
├── pages/              # 独立 HTML 页面存放目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库首页，提供页面列表或导航链接。
- `pages/`：存放所有独立 HTML 页面，可自行组织子目录。
- `assets/`：公共资源文件，供各页面引用。

---

## 页面归档说明

- 每个 HTML 页面应保持独立、完整，包括所需的样式与脚本（可通过 `<link>` 或 `<script>` 引用 `assets/` 中的资源）。
- 页面之间无耦合关系，可单独访问。
- 建议页面文件名使用英文小写、连字符分隔，便于 URL 访问。
- 如需更新或新增页面，直接向 `pages/` 目录添加文件即可。

---

## 使用方式

1. 克隆或 Fork 本仓库。
2. 在 `pages/` 目录下添加你的 HTML 文件。
3. 修改 `index.html` 更新页面列表链接（如有）。
4. 提交并推送至 GitHub，启用仓库的 Pages 功能即可在线访问。

> 若使用 GitHub Pages，默认访问地址为：  
> `https://<用户名>.github.io/resource-index-app/`
